# 发现与决策

## 需求
1. 采用Python开发，代码遵循生产规范，程序整体稳定性强
2. 自动筛查剔除无效、空白以及损坏文件
3. 文件依照文件名数字补零规则排序，保证排列顺序准确无误
4. 统一设置固定A4画布尺寸，各类规格页面居中摆放，小页面自动留白，大页面等比例适配，保证内容完整无变形
5. 采用流式合并方式处理大体积文件，分批加载内容，控制内存占用，防止程序卡顿崩溃
6. 配备文件防覆盖机制，输出前校验文件是否存在，已有文件则作出提醒，禁止直接覆盖保存
7. 覆盖各类运行异常捕获机制，应对路径、权限、文件损坏、磁盘空间不足等问题，程序运行稳定不闪退
8. 运行结束自动释放各类资源，有效规避内存泄漏问题
9. 处理好横屏竖屏的问题，不要出现字形是倒着的情况

## 研究发现
- 当前项目使用 pypdf/PyPDF2 库处理PDF
- PyPDF2 3.0.1 已安装在虚拟环境中
- 现有 merge.py 已实现基本的流式合并功能
- 现有 utils.py 已实现自然排序和覆盖保护功能
- 设计文档已创建：docs/superpowers/specs/2026-05-14-pdf-merge-improvement-design.md
- A4画布尺寸：595.28 x 841.89 点（210 x 297 毫米）

## 技术决策
| 决策 | 理由 |
|------|------|
| 创建独立的 pdf_processor.py 模块 | 职责分离，代码清晰，可维护性好 |
| 使用等比缩放+居中+留白方式适配A4 | 保持原始宽高比，内容完整无变形 |
| 自动矫正页面方向 | 确保文字方向正确，不出现倒置 |
| 逐文件处理 | 控制内存占用，防止程序卡顿 |
| 交互式确认覆盖 | 防止误覆盖已有文件 |
